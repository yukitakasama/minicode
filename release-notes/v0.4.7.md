# Claude Code Haha v0.4.7

这是 `v0.4.6` 之后的模型兼容性和桌面稳定性版本，重点新增 GPT-5.6 模型支持和页面内查找，同时修复长任务停滞、并发权限请求、后台任务生命周期、Claude Resume 会话发现、定时任务重复输出和跨平台窗口打开等问题。

相比 `v0.4.6`，本版本进一步收紧长时间运行会话的恢复链路和权限交互边界；OpenAI 官方登录可发现并选择 GPT-5.6 系列模型，Kimi K2.7 会自动满足其 thinking 要求，技能市场和会话活动面板也完成一轮视觉与交互优化。

## Highlights

- **GPT-5.6 模型支持**：OpenAI 官方登录新增 GPT-5.6、GPT-5.6 Pro 和 GPT-5.6 Codex 模型目录、选择和迁移支持，并补齐模型元数据与会话恢复链路。
- **长任务恢复更可靠**：长时间运行的 turn 在流式传输或 WebSocket 中断后能更准确地区分可恢复与已完成状态，避免界面长期停留在运行中或丢失最终 assistant 输出。
- **权限交互更安全**：同一会话中的并发权限请求会分别保留和响应；活跃 turn 期间禁止切换权限模式，避免运行时权限状态与界面选择不一致。
- **后台任务生命周期补齐**：后台任务支持更明确的停止、完成和活动面板状态同步，减少任务结束后仍显示 running 或无法停止的问题。
- **桌面工作流继续收敛**：新增 `Ctrl/Cmd+F` 页面内查找；Claude Resume 能发现桌面会话；从 Explorer 打开目录时不会再让已有窗口意外隐藏。

## Fixes

- 修复长时间运行的模型响应在 watchdog、重试、WebSocket 重连或 assistant commit 边界上可能卡住的问题，补齐可恢复事件和最终消息提交保护。
- 修复多个 permission / AskUserQuestion 请求同时到达时，后一个请求覆盖前一个请求或响应路由错误的问题。
- 修复会话仍在运行时可以切换权限模式的问题；现在 selector 会保持禁用并给出明确提示。
- 修复后台任务停止后状态未及时同步、活动面板缺少终止入口或任务结束后仍残留 running 状态的问题 (#990)。
- 修复桌面 API 定时轮询重叠导致重复请求、过期响应覆盖新状态或会话列表刷新异常的问题 (#993)。
- 修复定时任务完成输出被重复写入会话的问题 (#989)。
- 修复桌面主题设置影响 Claude CLI 输出主题的问题，应用主题与 CLI 配置重新隔离 (#987)。
- 修复 Kimi K2.7 在未显式开启 thinking 时请求失败的问题，该模型现在会自动使用必需的 thinking 配置 (#986)。
- 修复 Claude Resume 无法发现桌面创建的会话，以及 session 文件结束写入未及时 flush 的问题 (#988)。
- 修复从 Explorer 打开目录时复用窗口参数导致已有窗口被隐藏的问题 (#983)。
- 修复更新请求错误继承 provider proxy、安装更新时 quit-and-install 异常，以及 About 页面版本布局不正确的问题。
- 修复 Mermaid label 中斜杠等字符可能破坏节点形状，以及活动摘要过早显示省略号的问题。
- 修复会话活动入口显示无意义计数 badge、token 用量最长任务时长被截断的问题。

## Features and Improvements

- OpenAI 官方 provider 新增 GPT-5.6 系列模型目录、能力信息、模型选择分组和旧设置迁移。
- 新增页面内查找弹窗，可使用 `Ctrl+F` / `Cmd+F` 打开并在当前页面结果间跳转 (#974)。
- 技能市场重新整理搜索、筛选、卡片、详情、安装状态、安全提示、来源状态和免责声明布局，改善窄窗口与暗色主题下的可读性。
- 会话活动面板优化 SubAgent、后台任务和状态信息的展示，并移除容易误导的活动数量 badge。
- 欢迎语调整为“构建、调试和设计”，更准确地覆盖桌面工作台的使用场景。
- PR 质量门禁改进确定性测试路由、provider/chat contract、覆盖率和隔离环境，减少真实配置或外部网络对 CI 证据的影响。

## 安装

到本页面底部的 Assets 里，按系统和 CPU 架构下载对应文件：

| 平台 | 文件 |
|------|------|
| macOS Apple Silicon（M 系列） | `Claude-Code-Haha-0.4.7-mac-arm64.dmg` |
| macOS Intel | `Claude-Code-Haha-0.4.7-mac-x64.dmg` |
| Windows x64 | `Claude-Code-Haha-0.4.7-win-x64.exe` |
| Windows ARM64 | `Claude-Code-Haha-0.4.7-win-arm64.exe` |
| Linux x64 | `Claude-Code-Haha-0.4.7-linux-x86_64.AppImage` 或 `Claude-Code-Haha-0.4.7-linux-amd64.deb` |
| Linux ARM64 | `Claude-Code-Haha-0.4.7-linux-arm64.AppImage` 或 `Claude-Code-Haha-0.4.7-linux-arm64.deb` |

### macOS

双击 DMG 把应用拖入“应用程序”。如果 macOS 签名和公证验证通过，首次打开应直接走正常 Gatekeeper 流程。

如果仍提示“已损坏”或“无法验证开发者”，说明当前产物可能仍是 unsigned fallback 或本地 quarantine 状态未清理，可在终端执行：

```bash
xattr -cr /Applications/Claude\ Code\ Haha.app
```

也可以在“系统设置 → 隐私与安全性”里点“仍要打开”。

### Windows

双击 `.exe` 安装。SmartScreen 提示时点“更多信息”→“仍要运行”。

### Linux

AppImage：

```bash
chmod +x Claude-Code-Haha-0.4.7-linux-x86_64.AppImage
./Claude-Code-Haha-0.4.7-linux-x86_64.AppImage
```

deb：

```bash
sudo apt install ./Claude-Code-Haha-0.4.7-linux-amd64.deb
```

ARM64 机器把文件名换成 `...-linux-arm64.AppImage` / `...-linux-arm64.deb`。
